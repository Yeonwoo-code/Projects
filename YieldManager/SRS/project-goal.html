# 프로젝트 목표 (Project Goal)

# 프로젝트명: MATLAB 기반 GUI 애플리케이션 프레임워크

# 1. 프로젝트 개요

본 프로젝트는 MATLAB에서 Singleton 패턴을 적용한 GUI 애플리케이션 프레임워크입니다. 이 프레임워크는 App 클래스를 중심으로 구성되어, 애플리케이션의 중앙 관리자로서 역할을 하며, GUI_Manager 클래스를 통해 GUI 요소를 동적으로 관리하고 제어합니다.

# 2. 해결하려는 문제

MATLAB 환경에서 효율적이고 관리하기 쉬운 GUI 애플리케이션을 구축하는 데 있어, 반복적인 코드 작성 및 비효율적인 GUI 관리 문제를 해결합니다. 또한, 애플리케이션의 이벤트 처리 및 확장성을 고려하여, 코드 유지보수성과 확장성을 높이고, 다양한 GUI 이벤트를 동적으로 관리할 수 있는 구조를 제공합니다.

# 3. 주요 목표

  • **싱글톤 패턴(Singleton Pattern)**을 적용하여 애플리케이션 인스턴스가 하나만 존재하도록 보장.
  • 동적인 이벤트 등록 및 실행 기능을 통해, GUI 상에서 발생할 수 있는 다양한 이벤트를 효율적으로 관리.
  • App 클래스와 GUI_Manager 클래스의 분리로 모듈화된 코드 구조를 유지하고, 확장성 및 유지보수성을 고려.
  • MATLAB GUI 환경에서 사용자가 쉽게 접근하고 이해할 수 있는 직관적인 UI 구성.

# 4. 기능적 목표

  • 애플리케이션 관리: App 클래스는 싱글톤 패턴을 이용해 인스턴스 하나만 생성되며, 이를 통해 애플리케이션의 모든 제어가 이루어집니다.
  • GUI 관리: GUI_Manager는 사용자 인터페이스를 관리하고, 파일 메뉴 등을 동적으로 추가하여 GUI를 제어합니다.
  • 이벤트 관리: register()를 사용해 다양한 이벤트 핸들러를 등록하고, execute()를 통해 실행할 수 있게 합니다. 이벤트가 등록되지 않은 경우 경고 메시지를 표시합니다.
  • 애플리케이션 종료: Exit()을 호출하면 애플리케이션이 정상적으로 종료되며, GUI를 닫고 인스턴스를 제거하는 기능을 제공합니다.

# 5. 비기능적 목표

  • 확장성: 새로운 기능을 추가할 수 있도록, 이벤트 등록과 실행 메커니즘을 확장 가능한 구조로 설계합니다.
  • 유지보수성: 객체 지향적 설계를 채택하여, GUI와 핵심 애플리케이션 로직을 분리하고, 코드의 유지보수성을 높입니다.
  • 성능: 이벤트 관리에 containers.Map을 사용하여, 검색 속도 최적화와 효율적인 이벤트 관리를 목표로 합니다.
  • 사용자 경험: GUI는 직관적으로 구성되며, 사용자에게 친숙한 인터페이스를 제공합니다. 종료 시 적절한 메시지를 표시하여 사용자에게 명확한 피드백을 제공합니다.
