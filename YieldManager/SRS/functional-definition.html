# 기능 정의 (Functional Requirements)

# 2.1 애플리케이션 관리 (Singleton)

  • App 클래스는 싱글톤(Singleton) 패턴을 사용하여 단일 인스턴스만 생성될 수 있어야 합니다.
  • getInstance()를 호출하면 기존 인스턴스를 반환하며, 존재하지 않는 경우 새로운 인스턴스를 생성해야 합니다.
  • App 클래스의 생성자(App())는 private으로 설정하여 외부에서 직접 인스턴스를 생성하지 못하도록 제한해야 합니다.

# 2.2 GUI 관리

  • App 클래스는 GUI_Manager 인스턴스를 View 속성을 통해 관리해야 합니다.
  • GUI_Manager는 figure를 생성하여 애플리케이션의 GUI를 담당해야 합니다.
  • GUI_Manager는 **파일 메뉴(File Menu)**를 생성하고, 동적으로 등록된 이벤트를 실행할 수 있어야 합니다.

# 2.3 이벤트 등록 및 실행

  • register(item_name, handler)를 통해 이벤트 핸들러를 동적으로 추가할 수 있어야 합니다.
    • item_name: 이벤트 이름 (예: "Exit", "Open", "Save")
    • handler: 실행될 함수 핸들
  • execute(item_name)을 호출하면 등록된 이벤트를 실행할 수 있어야 합니다.
  • execute() 실행 시 등록되지 않은 이벤트를 요청할 경우 "이벤트가 등록되지 않음" 경고 메시지를 출력해야 합니다.

# 2.4 애플리케이션 종료

  • Exit()을 호출하면 애플리케이션을 정상적으로 종료해야 합니다.
  • 종료 시:
    • GUI_Manager에서 Figure를 닫아야 합니다.
    • App 인스턴스를 삭제해야 합니다.
    • MATLAB 작업 공간에서 App 인스턴스를 완전히 제거해야 합니다.
