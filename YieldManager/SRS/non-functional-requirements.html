# 3. 비기능 요구 사항 (Non-Functional Requirements)

# 3.1 확장성

  • 추가적인 이벤트 및 UI 요소를 쉽게 등록하고 확장할 수 있도록 설계되어야 한다.
  • register()와 execute()를 활용하여 동적으로 새로운 기능을 추가할 수 있어야 한다.

# 3.2 유지보수성

  • **객체 지향적 설계(OOP)**를 적용하여 코드의 모듈성을 유지해야 한다.
  • GUI와 핵심 로직을 분리하여, UI 변경이 핵심 애플리케이션 로직에 영향을 미치지 않도록 해야 한다.

# 3.3 성능

  • containers.Map을 활용하여 이벤트 관리의 검색 속도를 최적화해야 한다.
  • 불필요한 객체 생성을 방지하기 위해 Singleton 패턴을 적용해야 한다.

# 3.4 사용자 경험(UI/UX)

  • GUI 종료 시 사용자에게 메시지를 표시할 수 있어야 한다 (disp('GUI 종료됨')).
  • GUI 요소는 직관적으로 구성되어야 하며, 최소한의 학습 곡선으로 사용할 수 있어야 한다.
